Tiny Language Model Guided Flow Q Learning for Optimal Task
Scheduling in Fog Computing
Bhargavi K 1,*
 and Sajjan G. Shiva 2,*
1 Department of CSE, Siddaganga Institute of Technology, Tumakuru 572103, Karnataka, India

## §2 Department of CS, University of Memphis, Memphis, TN 38152, USA

* Correspondence: bhargavik@sit.ac.in (B.K.); sshiva@memphis.edu (S.G.S.)
Abstract
Fog computing is one of the rapidly growing platforms with an exponentially increasing
demand for real-time data processing. The fog computing market is expected to reach USD
8358 million by the year 2030 with a compound annual growth of 50%. The wide adaptation
of fog computing by the industries worldwide is due to the advantages like reduced
latency, high operational efficiency, and high-level data privacy. The highly distributed and
heterogeneous nature of fog computing leads to significant challenges related to resource
management, data security, task scheduling, data privacy, and interoperability. The task
typically represents a job generated by the IoT device. The action indicates the way of
executing the tasks whose decision is taken by the scheduler. Task scheduling is one of the
prominent issues in fog computing which includes the process of effectively scheduling
the tasks among fog devices to effectively utilize the resources and meet the Quality of
Service (QoS) requirements of the applications. Improper task scheduling leads to increased
execution time, overutilization of resources, data loss, and poor scalability. Hence there is a
need to do proper task scheduling to make optimal task distribution decisions in a highly
dynamic resource-constrained heterogeneous fog computing environment. Flow Q learning
(FQL) is a potential form of reinforcement learning algorithm which uses the flow matching
policy for action distribution. It can handle complex forms of data and multimodal action
distribution which make it suitable for the highly volatile fog computing environment.
However, flow Q learning struggles to achieve a proper trade-off between the expressive
flow model and a reduction in the Q function, as it relies on a one-step optimization
policy that introduces bias into the estimated Q function value. The Tiny Language Model
(TLM) is a significantly smaller form of a Large Language Model (LLM) which is designed
to operate over the device-constrained environment. It can provide fair and systematic
guidance to disproportionally biased deep learning models. In this paper a novel TLM
guided flow Q learning framework is designed to address the task scheduling problem
in fog computing. The neutrality and fine-tuning capability of the TLM is combined with
the quick generable ability of the FQL algorithm. The framework is simulated using the
Simcan2Fog simulator considering the dynamic nature of fog environment under finite
and infinite resources. The performance is found to be good with respect to parameters
like execution time, accuracy, response time, and latency. Further the results obtained are
validated using the expected value analysis method which is found to be satisfactory.
Keywords: fog computing; tiny language model; flow Q learning; service placement;
optimal solution

1. Introduction
Fog computing is a popular distributed computing environment which brings com-
putation and storage closer to the fog nodes. The proximity of the data towards the fog
nodes enhances the processing efficiency by reducing the latency and response time. There
is an exponential growth in fog computing as the need for real-time data processing at
fog nodes is high. The fog computing market is expected to reach 8358 million dollars
by the year 2030 with a compound annual growth of 50%. It offers greater scalability by
seamlessly adding the fog nodes at the edge network, and the resources are provided on a
demand basis. Because of the distributed nature of the fog system, single point failures are
prevented, and the system is resilient to attacks. The improved coordination among the
fog nodes leads to quick data analysis at edge devices, location awareness, and improved
overall efficiency of the fog services [1,2].
The dynamic nature typically represents time-specific varying characteristics of the fog
computing system which affect the scheduling decisions. The tasks generated by the IoT
devices vary continuously in unpredictable ways. The resource availability of fog nodes in
terms of bandwidth, memory, and CPU utilization rate keeps fluctuating. Even the network
conditions are not constant. The queue length keeps updating as the system state changes
after every scheduling decision. The heterogeneity of the fog system represents unequal
computation capability among the fog nodes. The energy, power, and latency across fog
nodes also keep varying. Fog nodes are also spread across a wide geographic location, and
every fog node cannot have a global and static view of the requirements. Hence it is not
possible to formulate a single scheduling policy which is optimal for all heterogeneous
task requests. Managing this uncertainty is challenging and demands the design and
development of a fine-tuned machine learning model [3,4]. The machine learning model is
mainly required to handle the high-dimensional, time-varying, and partially observable
nature of fog computing. The simple heuristic solution is not feasible for fog computing
as it is dependent on fixed-decision logic, limited awareness of fog states, and the static
assumption of network characteristics. The machine learning model should understand
system behavior and mine nonlinear relationships among fog nodes. The generic form
of the machine learning algorithm suffers from poor exploration, slow convergence, and
unstable learning over the large state space environment. The language model fine-tuned
the machine learning algorithm by injecting the prior knowledge, including a guide during
exploration, and accelerating the convergence process [5,6]. The fine-tuned machine learn-
ing models are considered as promising frameworks to address the performance issues
in the fog environment like resource management, task scheduling, service placement,
resource provisioning, and many more.
Out of all performance issues, task scheduling is considered as one of the most difficult
tasks in an uncertain fog environment because of various factors like the arrival rate of
tasks, fluctuating resource demands, and unpredictable network conditions. As the tasks
are generated continuously, the workload demands fluctuate dynamically. The resource
requirements of tasks generated by the heterogeneous IoT devices keeps fluctuating. The
network congestion and failure of fog devices can cause significant damage to the perfor-
mance of the fog system [5,6]. As a result, the chances of formulating the improper task
scheduling decisions are high. Improper task scheduling policies are the policies that oper-
ate randomly by ignoring the current state of operation. This causes long-term side effects
through violation of Service Level Agreements (SLAs) and inefficient usage of resources.
Various kinds of improper task scheduling happen that include random assignment, greedy
assignment, threshold-based assignment, and round robin assignment. In random as-
signment, the task is randomly assigned to the computing fog node without taking into
consideration current load and latency sensitivity of the tasks. In greedy assignment, all

incoming tasks are assigned to the nearest fog node intentionally even though that node
becomes overloaded and other nodes are underloaded. In threshold-based assignments,
a fixed threshold value is predetermined based on which decisions are made despite the
fluctuations in the network and task requirements. In round robin assignment, the tasks
are distributed among the fog nodes sequentially with fixed time slots for processing, in
which the heterogeneous nature of the tasks and resources are completely neglected. Hence
there is a need to develop a task scheduler which can handle uncertainties and formulate
high quality task scheduling policies [7].
In literature, several classical heuristic algorithms and machine learning algorithms are
developed for task scheduling in fog environments. Heuristic algorithms are dependent on
the fixed assumptions which fail to operate in a real-time fog computing environment. The
task’s arrival pattern is usually bursty and resources availability changes continuously. The
predetermined rules do not learn from experience and lack online adaptation. They perform
manual fine tuning of parameters and assume all tasks have identical QoS requirements. As
the complexity of the rules increases there is a communication overhead and parameter fine
tuning becomes infeasible. The partial observability and uncertain execution pattern of the
fog environment weakens the scheduler performance. Hence, the learning base approaches
the need to be designed to learn through experience and make long-term decisions through
adaptability. Machine learning models directly capture nonlinear interaction between
tasks, fog nodes, and network. Through trial and feedback mechanisms the machine
learning algorithm becomes tolerant to uncertainty and gains the ability to generalize from
experience [8]
Flow Q learning (FQL) is one of the promising offline forms of reinforcement learning
techniques which models complex action distributions using flow matching policy. The
existing actor critic framework is updated using flow matching policy and does not require
any form of sampling strategy. The flow policies are basic, and maximum value is reached
using recursive backpropagation. The core idea is to use a one-step policy along with
the flow policy with behavioral cloning to maximize the Q value. The performance of
the FQL is better than the Gaussian policies as it uses weighted regression, recursive
backpropagation, and rejection sampling. However, FQL performance suffers because of
overestimation of action values and is very sensitive to hyperparameter values. It also
struggles to achieve a proper trade-off between exploration and exploitation. The size of
the Q table grows exponentially and takes a longer time to converge to an optimal solution
due to slow updates of Q values. It also struggles with poor reward shaping and heuristics
that are not available for sampling the state and action pairs [9,10]. It becomes difficult for
the FQL algorithm to learn from scratch about complex dynamics of the fog computing
environment, so Tiny Language Models (TLMs) provide guidance to FQL in a structured
manner which accelerates the speed to learning and stabilizes it quickly.
TLMs are potential lightweight alternatives to Large Language Models (LLMs). The
main motivation to select a TLM is that it can operate in a resource-constrained environ-
ment. It also offers low inference overhead and allows for near-edge deployment. It is
made up of fewer parameters which are properly fine-tuned and can deliver high-level
performance using strategies such as byte-level tokenization, pooling, and weight training.
The TLMs are better compared to generic Large Language Models, deep neural networks,
heuristic models, and centralized cloud approaches. The generic language models include
Large Language Model Meta AI (LLaMA), Generative Pre-trained Transformer (GPT),
and Bidirectional Encoder Representations from Transformers (BERT). The deep neural
networks include Multi-Layer Perceptron (MLP), Convolutional Neural Network (CNN),
and Q learning. Heuristic approaches include the Genetic Algorithm (GA), Particle Swarm
Optimization (PSO), Ant Colony Optimization (ACO). Centralized cloud approaches in-

The heuristics of TLM speed up the learning capability of the FQL agent and recover

In this paper a novel TLM guided FQL framework is designed for the fog computing
environment. The FQL enriched with the heuristics of the TLM can formulate optimal
task scheduling policies even from imperfect data by considering the suggestions from
generative artificial intelligence models. Immediate shaping of Q values is carried out
according to the heuristics of TLM which enhances the reward shaping mechanism [13].
The unnecessary explorations over the state space are prevented through pre-sampling and
post-sampling of the state space under the guidelines of the TLM [14,15].

• Performing a literature survey over the recent existing works and summarizing the

• Design of a novel TLM guided FQL framework to perform task scheduling.
• Expected value analysis of the TLM guided FQL framework considering finite and

• Simulation of the TLM guided FQL framework using Simcan2Fog simulator consider-

2. Related Work
In this section some of the potential recent papers on task scheduling for fog computing
are discussed and their limitations are identified.
Choppara et al. presents an adaptive form of the task scheduling framework for
fog computing by combining the federated deep Q network with the K-means clustering
algorithm [16]. Each of the fog nodes in the fog environment is made capable enough for
training the local deep Q network model. The experience of each of the fog nodes will be
merged to formulate a global model without the need to share any sensitive information.
The federated nature of learning provides a self-optimizing solution which evenly spreads
the learning among all fog nodes. The priorities are assigned for the incoming tasks
considering the two parameters of execution time and deadline for completion using the
K-means clustering algorithm. Care is always taken such that higher priority tasks are
executed by minimizing the rejection rate and Service Level Agreement violations. The
model continuously upgrades itself and adapts to a dynamically changing environment
through iterative learning. The localized form of learning incorporates constant learning
among local deep Q learning agents which constantly update the task scheduling policies
https://doi.org/10.3390/a19010060
Algorithms 2026, 19, 60 5 of 28
considering both task and environment characteristics. The experimental evaluation over
the larger dataset ensures that makespan time is reduced and throughput is enhanced.
Since distributed task scheduling is performed, the framework acts as a flexible solution for
latency-sensitive applications. For a larger network like fog, decentralization themed task
scheduling enhances the system scalability and reduces the communication overhead but
demands for high synchronization. Prioritized experience replay is performed in which
higher priorities enable the model to focus over the most informative experience. However,
the framework demands frequent updates from the sender and receiver models which
makes it expensive. The deeply nested Q network often causes instability of the network
due to frequent updating of the Q value and higher sampling efficiency over the large state
space environment.
Choppara et al. discuss a flexible task scheduling framework based on the deep
deterministic form of policy gradient (DDPG) for fog computing [ 17]. The DDPG is
dynamically adaptable for continuous and large state space environments in which the
task scheduling policies adapt themselves according to real-time demands of the tasks in
terms of resources. The DDPG basically follows the actor critic learning model in which the
actor learns the policy and the critic evaluates the action taken using the Q function value.
The experimental results reveal that satisfactory performance is achieved by reducing
the makespan time and increasing the fault tolerance. The data processing operation
is localized which reduces the bandwidth requirement and enhances the efficiency of
data management across the fog nodes. The priority score is calculated for each of the
tasks considering the key parameters like resource requirement, duration of execution,
task deadline, and dependency for the tasks. Critical applications and delay-sensitive
applications are executed first based on the priority score. After prioritization, the tasks
are classified into three categories which include sequential tasks, parallel tasks, and
dependency-oriented tasks. For sequential tasks, an execution order will be defined which
is well-suited for data pipeline applications. The parallel tasks are executed on a certain
set of virtual machines which are mounted over the fog nodes. For dependency-based
tasks, logical consistency is ensured in the workflow which demands to satisfy the specific
requirements prior to execution. However, the framework is subjected to a divergence
problem due to high variance encountered in the estimation of the Q value. Also, a tendency
to end up in local optima is high due to high sensitivity towards hyperparameters and
sparse reward in the deterministic fog computing environment.
Sudheer et al. discuss a methodology for task scheduling in fog computing by com-
bining priority awareness approach with the simulated annealing mechanism [ 18]. The
simulated annealing algorithm begins with the initialization of the state space with sub opti-
mal solutions. Later the optimality of the solution will be improved by decreasing the value
of the temperature variable. The random task plan is assigned for each of the fog nodes at
first. Afterwards a small tweak in the task plan is performed by considering the parameters
like energy consumption, cost of execution, completion time of task, and priority assigned
for the tasks. This leads to the generation of neighbors and a controlled exploration of the
state space is performed. The small tweak performed affects only two workloads: the old
node which was executing the workload and the new node to which the workload was
transferred. Initially the temperature is set to be high, but the algorithm slowly reduces the
temperature value through the hierarchy to converge with the best potential solution in
a fine-grained approach. The scheduling policies are improved gradually by computing
the cost for each of the small changes in an iterative manner. During the annealing phase,
higher weightage is given for exploring the new solutions instead of revisiting the existing
solutions. The movement is carried out in an uphill manner by accepting all the worst
forms of solutions which allow the exploration of state space in a wider manner. The proper

trade-off is achieved between exploration and exploitation through tough optimization
process. During the cooling phase, the gradual cooling of temperature is achieved to
arrive at an ideal solution while keeping the search area open wide enough for traversal.
Finally, algorithms stop when the temperature falls below the threshold or the k number
of iterations are completed. The performance results show that the high priority tasks are
processed early using a priority aware penalty function. The solution space is approached
in a very strict and controlled manner this prevents large shifts leading to a local optimal
solution. However, the algorithm is completely stochastic in nature which leads to different
kinds of outcome at every run. The cooling process is slow and requires more epochs of
training compared to the traditional spring embedders technique.
Asad et al. presents a novel task scheduling framework for a cloud and fog environ-
ment using the Harris Hawk Optimization (HHO) technique [19]. Task scheduling is an
important issue as it is difficult to find appropriate resources for the execution of the tasks.
Scheduling the client task to fog nodes means tasks can be executed with less delay but
it deals with more energy consumption. Meanwhile scheduling the tasks onto a cloud
server consumes less energy but more delay is incurred. Hence it is essential for the task
scheduling algorithm to consider all the factors including energy and delay before arriving
at scheduling decisions. The HHO technique mimics the cooperative hunting strategy of
the Harris Hawk. It mainly works in two phases: exploration and exploitation. During
the exploration phase the location of prey is located by the Hawk. Similarly, during the
exploitation phase, all possible attack strategies are executed over the prey. At first the
arrival rate of tasks and energy consumed for escaping of prey are initialized. The task is
assigned to fog nodes based on the factors like the Quality of Service (QoS) requirements of
tasks and location of the data source. The diversity in the available solution is determined
using inverse mutation operation. The best solution out of all available solutions is stored in
the repository as the non-dominated solution. Through epochs of training the new solution
replaces the old solution based on the value of crowding distance. The soft besiege phase
represents the start of the attacking strategy in which the energy level of the prey is depleted
using the encircling mechanism. Similarly, the hard besiege phase performs encircling of
the prey in a zigzag manner. The deep dive operation of the Hawks is performed using
the Levy flight procedure. Whenever the prey becomes fatigued an effective attack will
be launched to execute the surprise pounce. The process is repeated until termination is
reached and the best solution available at the search agent is considered as the optimal
scheduling policy. The experimental results indicate that there is a significant reduction in
energy consumption and delay. Guaranteed performance is achieved by achieving proper
balance between intensification and diversification phases. However, random parameters
are involved in the exploration and exploitation phases which cause negative effects over
the convergence speed and lead to higher computational cost.
Seyed et al. perform the scheduling of a bunch of tasks in fog computing using the
discrete cuckoo search algorithm [20]. First the incoming client applications are broken
down into many independent tasks. A centralized broker node is maintained which is
responsible for maintaining the capacity of fog nodes along with their failure rates. The
broker node is responsible for performing load balancing and allocating the task onto the
most reliable fog node. The myriad permutation policy is followed by delivery of different
varieties of solutions to the incoming client requests. The typical cuckoo search algorithm
is converted into hybrid form by incorporating two procedures related to sorting and
crowding distance computation. Numerous local search operators are used to perform
exhaustive search over the solution space which consists of the worst solution and best
solution. The discrete search space of the fog is explored towards the efficient form of quest
permutations. The diverse novel exploration operation is invoked randomly using the

Levy flight search mechanism. The workflow is typically represented as a directed acyclic
graph; the exploration and exploitation operators are executed during the encoding stage.
The topological ordering of the tasks is determined to preserve the constraints among
the tasks. The main aim of the optimization is to generate new capable solutions over an
iteration of training. If the new solution generated is better than the previous solution,
then the new one is considered as the dominant solution. Here multiple objectives are
taken into consideration while delivering the solution. The solutions are divided into
two sets that are an inferior set of solutions and superior set of solutions using Pareto
ranking values. The superior set of solutions focuses on performing multiple objectives
optimization. Whereas the inferior set of solutions focuses on finding the single objective
optimization. The set of solutions which do not compete are assigned to first rank and
known as non-dominated solutions. The first rank solutions are removed from the current
solutions, and the procedure is repeated for the remaining solutions. Likewise, second and
third ranking solutions are represented. All solutions generated are sorted based on the
number of objectives and crowding distance value. The solution with higher crowding
distance value is considered as beneficial because it exhibits a high rate of diversity. The
experimental results showcase that significant improvement is observed with respect to
reliability and makespan time in all varying scenarios of workload conditions. However,
here the scheduler is not aware of the user’s Quality of Service requirements, which
increases the monetary cost of operation and the scheduler fails to achieve the desired QoS.
Seohang et al. discussed the basic working of flow Q learning algorithm [ 21]. It
projects the flow matching policy to represent the complex action distribution in a large
state space environment. The flow model is based on the stochastic differential equation
which allows for faster training and converges with better quality solutions. Three im-
portant benefits offered by flow Q learning are as follows: It uses a reparametrized policy
gradient which perform effective exploration of state space and prevents the unstable and
expensive backpropagation mechanism. Second, the one-step policy is generated as the
output which overcomes the inherent limitations of iterative flow generation process. Third,
the implementation is easy and fine tuning is performed at optimal pace. It is composed of
only one major hyperparameter and does not demand fine tuning in a noisy environment.
However, flow Q learning suffers from training instability and poor generalization capa-
bility under dynamic scenario. It consists of many sensitive parameters which need to be
fine-tuned and exhibits poor scalability due to the composition of a large input space. To
overcome exploration inefficiency and solve the credit assignment problem, poor guidance
is required.
Fuxue et al. discuss the need for LLM guidance to the traditional reinforcement
learning algorithm [22]. The reinforcement learning algorithm cannot perform high-level
planning and low-level task execution simultaneously. The LLM is responsible for de-
composing larger tasks into a sequence of manageable subtasks in an interactive state
space heterogeneous environment. During training even the intermediate form of subgoals
are associated with the rewards and fine-grained feedback is provided to accelerate the
convergence rate and perform structured exploration of the large state space environment.
The chain of thought strategy is incorporated which allows the LLM to quickly update to
changing dynamics of the environment state space. The exploration inefficiency in sparse
reward setting is overcome by conditioning the subgoals and reward functions. The LLM
guided Q learning operates using generic prompts which prevent frequent retraining of
the model. The reward is scaled as per the completion of subgoals, and more rewards are
obtained through faster scaling. However, LLM-enabled reinforcement learning algorithms
cannot be deployed for the fog environment, as LLMs are composed of billions of parame-

ters and demand a large amount of memory. The output of LLM is often hallucinated and
even invalid which makes it incompatible with the larger scale edge deployments scenarios.
To summarize, most of the existing works exhibit the following drawbacks:
• The deep neural network is often stuck in learning instability over a large state space
environment which leads to learning instability and converges with non-desirable
policies [16].
• The deterministic policy gradients generated by the Q learning agent are extremely
sensitive towards the selection of hyperparameters. They also suffer from sparse
reward problems [17].
• Metaheuristic algorithms are extremely costly over the distributed form of a hetero-
geneous environment. Quick scheduling decisions are taken using heuristic values
which leads to errors in task scheduling decisions [19,20].
• The rule-based algorithms usually fail to capture the uncertainty of the devices, work-
load, and network condition. Converges to suboptimal solutions due to improper
balance between the exploration and exploitation activities [3].
• Supervised and unsupervised machine learning algorithms suffer from poor search
progress and weaker efficiency. They are often subjected to overfitting problems and
suffer from generalization issues due to high variance and learning instability [5,6].
• Conventional flow Q learning algorithms suffer from credit assignment problems and
poor exploration capability [21].
• Traditional LLM guided reinforcement learning is subjected to the sparse reward
problem and struggles to perform multistage scheduling in long horizons. Often ends
up with invalid outputs and inefficient utilization of resources [22].
3. System Model
The fog computing system model is composed of a set of tasksT = {T1, T2, T3, T4, . . . ,Tm}
and a set of fog nodesFN = {FN1, FN2, FN3, FN4, . . . ,FNn}. Each fog node is made up of a
collection of virtual machinesVM = {VM 1, VM 2, VM 3, VM 4, . . . ,VM k}. Each of the tasks is
described using five attributes: task durationTD, resource requirementsRR, deadline proxim-
ity DP, dependency countDC, and impact factorIF, i.e., Ti =< TD, RR, DP, DC, IF >. The
TD is the estimated execution time of the tasks on any fog node. TheRR is the computational
resource requirement of the task to be executed. TheDP indicates the deadline sensitivity of
the task. The DC represents the number of tasks that are dependent on the execution of the
current task. The impact factorIF is the composite measure of the severity of the poor schedul-
ing decisions. For example: For healthcare application a sample attributes for a particular
task includesTi =< TD = 10ms, RR = 200 CPUs, DP = Less, DC = 3 Tasks, IF = High >.
For a small execution time, theIF value must be lesser for critical applications like healthcare
and military in which improper scheduling results in severe consequences and ends up in
poor Quality of Service (QoS). Similarly , each of the virtual machines represents the current
workloadC(WL) and capacity of the CPU,CP(CPU), for all virtual machines in the fog node,
i.e., VM i =< C(WL), CP(CPU) >. The task scheduling problem is typically represented as
a Markov Decision Process (MDP),MDP(Ti) =< S, A, P, R, γ >, where S is the state space,
A is the action space, P is the transition probability ,R is the reward function, andγ is the
discount factor which achieves balance between the immediate reward and future reward.

The incoming tasks are arranged in a queue; the arrival and departure rate of the tasks
are managed by a task scheduler. During FQL the training of flow policy is performed using

The mathematical definition for the evaluation metrics (EM) considered for evaluation
of the framework is defined as follows. The reason behind the selection of these evaluation
metrics is that they are the benchmark metrics followed worldwide for fog deployment.
They basically represent the main objectives and constraints of a fog computing environ-
ment. In which the task requirements are dynamic, resources are less, but still there is a need
to provide best Quality of Service (QoS). Cost metrics represent the economic efficiency
of the task scheduler. Energy consumption metric is a critical evaluation metric because
the scheduler needs to operate in a resource-constrained environment. SLA violation rate
is important for delay-sensitive applications to meet the deadline and latency constraints.
Scalability indicates the improved performance of the scheduler with the increasing number
of tasks and fog nodes. Reliability is the ability of the scheduler to function even under fog
nodes failure and network disruptions [23].

4. Proposed Work
The high-level architecture of TLM guided FQL framework is shown in Figure 1.
The architecture is composed of two main components that are the Tiny Language Model
(TLM) and flow Q learning (FQL). TLM is a mini version of the Large Language Model
(LLM) which has few parameters and can perform with lesser computational resources.
The training of TLM is performed by a pre-trained model through knowledge distillation.
Pruning is performed to remove unnecessary parts of the model and model weights are

updated through quantization. Deploying TLM on edge devices allows ease of interaction
among edge devices, analysis, and localization of the data. It begins with the preparation
of its own data through model pretraining. The model is built as an open GPT-2 basic
model whose parameters vary from 30 M to 124 M. Fine tuning of the data is performed
through downstream embedding of the sensed tasks. The trained model is converted into
an optimized model to provide guidance to the FQL.

The flowchart representation of the TLM guided FQL is given in Figure 2. The FQL
considers the heuristics of TLM to generate flow policies through precise modeling of the
complex action distributions in the fog domain. Instead of performing random initialization
of the parameters of the Q learning agent, the TLM are prompted to generate state, action,
and Q values based on the high-level description of the tasks and fog nodes. This provides
a promising starting point for the agent and only significant heuristics are added to the
Q value update function which leads to execution of the best action at any given state
of operation. The diffusion policies are trained to maximize the learned value function
of the Q learning agent. behavior cloning is a form of supervised learning which trains
the Q learning agent by mimicking the expert demonstrations. The sampling efficiency is
improved through behavioral cloning, and it is also less subjected to compounding errors.
Traditional Q learning algorithms are trained using multi-step policies which suffer from
performance degradation as it is highly sensitive towards the number of steps chosen.
This problem is overcome here using a one-step policy which enables direct one-step
action prediction from the state through the flow matching mechanism. Direct mapping is
performed from state to action, and the necessity of a separate iterative process is eliminated
by the learning velocity field which maps noise to action. The performance of the FQL
is higher compared to the Gaussian model and diffusion model as it can easily learn
multimodal action distributions in a complex environment. Even the poor heuristics of the
TLM are handled effectively by the Q learning agent through environment feedback and it
is ensured that the task scheduling policies are unbiased and optimal.

The fog computing system sate space is defined by five attributes, i.e., FC_SS =
{RFN , QFN , EFN , LFN , TFN }, RFN is the resources at the fog nodes, QFN is the queue length
at each of the fog node, EFN is the energy of the fog node, LFN is the network latency, and
TFN is the attributes of the tasks. The concrete scheduling action is taken for each of the
tasks, ai = {Execute , O f f load to cloud , Delay /reject }, by considering the heuristic scores
of the actions generated by TLM. The input for TLM includes the characteristics of tasks,
fog nodes, and the network condition. The state to encoder converts the numerical input
into textual tokes. For example, if CPU utilization is greater than 80 percent it is considered
as high load. TLM outputs a small set of action preference tokens. The deterministic action
mapping subroutine is responsible for converting the tokens of TLN into heuristic scores.
The heuristic score ranges between −0.5 and +1.0.
The FQL is mainly composed of three components, i.e., criticQ∅(s, a), behavior cloning
flow policy µθ(s, z), and one-step policy µw(s, z). The critic basically provides learning
signal via a gradient target for the flow policy generated by the agent. Bellman consistency
is ensured by the critic by precisely estimating the future retunes. The behavior cloning
flow policy trains the flow policy to mimic the action of the dataset and it becomes the
starting point of the agent. It also teaches the flow model to map noise to action. After
performing behavior cloning the Q learning agent improves itself beyond the initialized
dataset. The one-step policy takes the action immediately from a state without the need for
any iterations. However, multiple steps of transformation are executed within to select a
particular action as the optimal one. Using one-step policy the TLM_FQL parses through
the tasks set by considering only the present Q state. This makes it greedy and leads to
immediate scheduling decisions with multi-step planning.

6. Results and Discussion
The proposed TLM_FQL framework is compared with three of the recent existing
works, i.e., DDPG [17], DCS [20], HHO [19], and FQL [21]. The task scheduling environment
is composed of two layers: the IoT devices layer and fog layer. The IoT devices layer is
responsible for generating the tasks which follow a stochastic arrival pattern. Similarly,
the fog layer is composed of multiple fog nodes with varying processing capacity which
process the tasks which are latency insensitive. If sufficient resources are not available,
then they will be processed at the cloud layer with an unlimited number of resources. The
TLM_FQL framework interacts with the fog environment based on the current state of
operation. The current state is determined by considering the current dynamics of tasks,
resources, and network conditions. The local task scheduling actions are chosen by the
https://doi.org/10.3390/a19010060

FQL considering the feedback of the TLM. The fog environment updates its state after the
action and provides feedback over the evaluation metrics.

the fog computing simulator [26,27]. It incorporates the real-time behavior of the network,
hardware modeling, and cloud orchestration. It allows precise representation of network
behavior considering topology management, obstacles detection, transport protocol, and
traffic control mechanism. The Simcan2Fog simulator is an extension of the Simcan2Cloud
simulator which provides higher end simulation features like virtualization, data cen-
ter operations, queuing mechanism, and cost evaluations. The simulation parameters

7. Conclusions
The paper presents a novel Tiny Language Model-inspired flow Q learning framework
for task scheduling in fog computing. The heuristics of TLM are provided as guidance
for Q learning to perform flow models and behavior cloning of policies through one-step
optimization. The structure priors of TLM reduce the tendency of overestimation of the Q
value and make the bootstrapping process smoother. Generalization of the behavior cloning
is easy through demonstrations and lowers the risks of hallucinations. The experimental
results demonstrate that the performance of the TLM_FQL is good with respect to metrics
like cost, energy consumption, SLA violation, and scalability considering small and big fog
computing scenarios. The results are further validated using expected value analysis and
results are found to be satisfactory. For future work, trajectory will be used to properly
represent the state, action, and reward of the learning agent. Pruning of unsafe trajectories
will be carried out to enhance the robustness of the scheduling policies and improve
learning efficiency. The noise in the state space is removed through selective experience
replay and produces policy reliability under adverse operating conditions. The integration
of safe reinforcement learning along with the trajectory pruning ensure faster learning and
prevents the catastrophic failures of the tasks.

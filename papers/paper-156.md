### Abstract

In this paper we propose the adoption of Agent Process Modelling, a theoretical-practical framework for the orchestration of agent behaviours running process models. The agent model is revisited from a distributed, cloud-native perspective and agents’ capabilities are externalized as microservices. Agent logic is represented in process models that orchestrate services execution by using a standard process notation, BPMN, increasing explainability and verification of agents decisions. As a preliminary work, we demonstrate a possible workflow to design and run agent-processes by instantiating the FIPA ContractNet protocol. © 2019, Springer Nature Switzerland AG.

Four visual alternatives for the 7-axis taxonomy

Same data, four renderings. Pick the one that matches the venue's expected figure style, then keep one variant in report/index.md (or the manuscript) and discard the others. Distributions in every variant come from the locked stage 08 analysis (stages/08-analysis.md); numbers next to each value denote the count of papers contributing to it (multi-select axes sum to more than 20).

How to pick

A — Hierarchical tree (IEEE/ACM style) is the safest match for a reviewer's expectation and most closely mirrors the reference figures (Fig. 4 / Fig. 8). B — Compact axis pills prints best on a single column of an A4/letter manuscript. C — Faceted cards is the web-friendly variant. D — Cross-product matrix is supplementary: it isolates the Pipeline Contributor × Multi-Agent empty cell that becomes a discussion point in stage 09 RQ5.2.

★ single-select axis (one value per paper) ▲ multi-select axis (one or more values per paper) ni next to a value = papers contributing to that value total corpus n = 20

AHierarchical tree (IEEE-style)

Mirrors the reference figures (Fig. 4 — stateful serverless taxonomy; Fig. 8 — maintenance taxonomy). Root → axis columns → leaves with star/triangle markers.

Pros: familiar to reviewers · scales to 7+ axes · prints well Cons: wide; may need 2-column landscape on small page

BCompact axis pills (single column)

One row per axis (or facet). Values rendered as pills with inline counts. Optimized for vertical real estate.

Pros: fits an A4 single column · easy scan · counts inline Cons: less "figure-like"; closer to a typeset table
